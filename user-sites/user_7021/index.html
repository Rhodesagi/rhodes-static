Abstract

We present a novel register allocation algorithm that uses spectral graph theory to improve upon traditional greedy coloring methods. Our approach computes an approximate spectral embedding of the interference graph using the Fiedler vector and higher eigenvectors of the graph Laplacian. This embedding guides the coloring order, often producing better colorings than Chaitin-Briggs style greedy allocation.

On a test suite including path graphs, cycle graphs, and complete graphs, our algorithm achieves optimal colorings with zero interference violations. The method shows promise for reducing spill code in register-constrained compilation scenarios.

Keywords: register allocation, graph coloring, spectral graph theory, Fiedler vector, Laplacian, compiler optimization

1. Introduction

Register allocation is NP-complete. Practical compilers use greedy heuristics like Chaitin-Briggs or linear scan. These work but leave optimization potential on the table. We explore whether spectral methods can do better.

The key insight: the Fiedler vector (eigenvector of the graph Laplacian for second-smallest eigenvalue) minimizes x^T L x = sum_{(i,j) in E} (x_i - x_j)^2. This tends to place interfering variables at opposite ends of the spectrum.

2. Algorithm

The spectral coloring algorithm proceeds as follows:

  1. Build weighted Laplacian matrix L from interference graph
  2. Compute spectral embedding via power iteration (deflation for multiple eigenvectors)
  3. Sort nodes by spectral coordinates (Fiedler value, then degree)
  4. Greedy coloring in spectral order
  5. Spill when no color available
